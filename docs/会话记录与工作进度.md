## 三、关键技术结论

### 3.1 模型说明

- **Faster R-CNN**：nuScenes 10类模型，PyTorch 导出 ONNX（2113节点），ResNet-50 FPN backbone
  - ONNX 路径：`src/faster_rcnn_ros/models/fasterrcnn_nuscenes_fixed_375x1242.onnx`
  - 10 个类：background, car, truck, bus, trailer, construction_vehicle, pedestrian, motorcycle, bicycle, traffic_cone, barrier
  - 输入：`image` [1,3,375,1242]，输出：`boxes[?,4]`, `labels[?]`(INT64), `scores[?]`
  - **注意**：labels 输出是 INT64，C++ 节点以 sizeof(int)=4 读取，小值时 little-endian 有效

- **YOLOv8n**：ultralytics 官方预训练，640×640，COCO 80类
  - 权重：`yolov8n.pt`（已入库）
  - TRT 引擎：`yolov8n_trt.engine`（需在目标设备重建）

### 3.2 TRT 引擎构建

**当前最优引擎**：`faster_rcnn_375_maxn.engine`（85MB，FP16，MAXN 环境下构建）

```bash
# 完整命令（在 MAXN + jetson_clocks 锁频后执行）
/usr/src/tensorrt/bin/trtexec \
  --onnx=src/faster_rcnn_ros/models/fasterrcnn_nuscenes_fixed_375x1242.onnx \
  --saveEngine=src/faster_rcnn_ros/models/faster_rcnn_375_maxn.engine \
  --fp16 --memPoolSize=workspace:8192MiB \
  --minShapes=image:1x3x375x1242 \
  --optShapes=image:1x3x375x1242 \
  --maxShapes=image:1x3x375x1242
# 耗时约 12 分钟
```

**INT8 限制**：TRT 8.5.2 中，Faster R-CNN 的动态 NMS 输出触发 `divUp(0, blockDim)` 断言失败，无法进行 INT8 PTQ 校准，是 TRT 8.5 已知限制。TRT 10.x（JetPack 6.x）理论上修复。

### 3.3 ROS2 节点关键配置

```xml
<!-- faster_rcnn.launch.xml 当前配置 -->
<param name="engine_path" value="$(find faster_rcnn_ros)/models/faster_rcnn_375_maxn.engine"/>
<param name="input_height" value="375"/>
<param name="input_width" value="1242"/>
<param name="threshold" value="0.5"/>
```

**启动命令**：
```bash
source /opt/ros/foxy/setup.bash
source install/setup.bash
LD_LIBRARY_PATH=/usr/local/lib:$LD_LIBRARY_PATH \
  ros2 launch faster_rcnn_ros faster_rcnn.launch.xml
```

### 待决策事项

#### 选项 A：升级 JetPack 6.1（推荐，中等工作量）
- **Ubuntu**：20.04 → 22.04，**ROS**：Foxy → Humble，**TRT**：8.5 → 10.3
- 潜在收益：FP16 提升 5-10% + **INT8 PTQ 可能解锁（28-35ms）**
- 风险：NVMe 启动设备需在 SDK Manager 中正确选择刷机目标
- **刷机后操作清单**：
  1. `git clone git@github.com:angqi1/fasterr-cnn.git ros2_ws`
  2. 按 DATASETS.md 恢复 SSLAD-2D 数据
  3. `source /opt/ros/humble/setup.bash && colcon build --symlink-install`
  4. 重建 TRT 引擎（见 3.2 节命令）
  5. 尝试 INT8 校准：`python3 build_int8_calibrated_engine.py`
  6. 验证 INT8 引擎精度：`python3 eval_precision.py`

#### 选项 B：换用 YOLOv8（推荐，低风险）
- **无需刷机**，立即可用
- 速度 8.7ms（MAXN 下约 5-6ms），精度全面超过 Faster R-CNN
- 需要新增 YOLOv8 ROS2 推理节点（或修改现有 C++ 节点）
- ONNX 导出：`python3 -c "from ultralytics import YOLO; YOLO('yolov8n.pt').export(format='onnx')"`
- DLA 部署：`trtexec --onnx=yolov8n.onnx --saveEngine=yolov8n_dla.engine --fp16 --useDLACore=0 --allowGPUFallback`

#### 选项 C：雷达+摄像头融合 Pipeline（终极目标）
- 推荐架构：YOLOv8 on DLA0（~12ms）+ 雷达点云 on GPU（~5ms）并行
- 预期端到端延迟：~12ms（>65fps）
- 参考报告：`docs/Jetson_AGX_Orin硬件优化与推理性能报告.md` 第 8 节
