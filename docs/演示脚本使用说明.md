# demo.sh / demo_presentation.py 使用说明

## 一、快速开始

```bash
cd /home/nvidia/ros2_ws
bash demo.sh
```

这一条命令将自动完成以下所有工作：
1. 检查 Python/TRT/OpenCV/ROS2 环境
2. 验证引擎文件和测试数据
3. 展示项目架构图
4. 对 6 张代表性图片运行实时 TRT 推理（FP16，约 40 秒）
5. 打印四配置精度对比表
6. 生成检测结果 Mosaic 拼图（`demo_mosaic.jpg`）
7. 生成自包含 HTML 培训报告（`demo_report.html`）

---

## 二、运行前准备

| 条件 | 说明 |
|------|------|
| 必须 | `install/faster_rcnn_ros/share/faster_rcnn_ros/models/faster_rcnn_500.engine` 存在 |
| 推荐 | `install/.../faster_rcnn_500_int8.engine` 存在（精度对比会更完整） |
| 推荐 | `test_images/kitti_100/images/` 有 100 张 PNG |
| 可选 | `test_images/compare_results/gt_eval_summary.json`（精度表需要） |
| 可选 | `sudo jetson_clocks`（GPU 锁频，推理更稳定） |

### 检查引擎是否存在

```bash
ls -lh install/faster_rcnn_ros/share/faster_rcnn_ros/models/*.engine
```

如果不存在，先构建：

```bash
# FP16 引擎（约 5 分钟）
python3 src/faster_rcnn_ros/models/build_engine.py \
    --height 500 --width 1242 --precision fp16

# INT8 引擎（约 8 分钟）
python3 src/faster_rcnn_ros/models/build_engine.py \
    --height 500 --width 1242 --precision int8
```

### 检查精度评估结果是否存在

```bash
cat test_images/compare_results/gt_eval_summary.json
```

如不存在，先评估（约 2 分钟）：

```bash
python3 gt_eval_all.py
```

## 三、命令行参数详解

### demo.sh

```bash
bash demo.sh [选项]
```

| 参数 | 含义 | 示例 |
|------|------|------|
| `--no-infer` | 跳过实时 TRT 推理，仅展示已有结果 | `bash demo.sh --no-infer` |
| `--stage N` | 只运行指定阶段（见下表） | `bash demo.sh --stage 3` |
| `--images N` | 实时推理的图片数量（默认 6） | `bash demo.sh --images 3` |
| `--threshold F` | 检测置信度阈值（默认 0.5） | `bash demo.sh --threshold 0.3` |
| `--output PATH` | HTML 报告路径（默认 `demo_report.html`） | `bash demo.sh --output /tmp/r.html` |
| `--no-clock` | 跳过 jetson_clocks 建议提示 | `bash demo.sh --no-clock` |
| `--help` | 显示帮助信息 | `bash demo.sh --help` |

### demo_presentation.py（直接调用）

```bash
python3 demo_presentation.py [选项]
```

参数与 demo.sh 相同，但没有 `--no-clock`。

## 四、演示阶段说明

| 阶段 | `--stage` | 内容 | 耗时 |
|------|-----------|------|------|
| 0 | 不可跳过 | 环境检查（Python/TRT/OpenCV/引擎/数据）| <1s |
| 1 | `1` | 系统架构图 + Faster R-CNN 两阶段流程 | <1s |
| 2 | `2` | FP16 引擎实时推理（6张，含 warmup） | ~45s |
| 3 | `3` | 四配置精度对比表 + 关键结论 | <1s |
| 4 | `4` | 检测结果 Mosaic 可视化 | ~2s |
| 全程 | `0`（默认） | 所有阶段 + 生成 HTML 报告 | ~2min |

## 五、典型使用场景

### 场景 A：完整培训演示（推荐）

```bash
sudo jetson_clocks          # 可选：锁频提升稳定性
bash demo.sh --no-clock     # 全流程演示
# 在浏览器打开报告
xdg-open demo_report.html
```

### 场景 B：讲师想先展示架构，再演示推理

```bash
# 第一步：只看架构
bash demo.sh --stage 1

# 第二步：运行推理（学员可观察实时输出）
bash demo.sh --stage 2

# 第三步：讲解精度指标
bash demo.sh --stage 3
```

### 场景 C：快速演示（无需等待推理）

```bash
bash demo.sh --no-infer
```

这将跳过 TRT 推理，使用预计算的图片展示结果，全程 <10 秒。

### 场景 D：对比不同阈值效果

```bash
# 展示高召回但多误检
bash demo.sh --threshold 0.25 --no-infer

# 展示精准但漏检较多
bash demo.sh --threshold 0.7 --no-infer
```

### 场景 E：只看精度数据

```bash
python3 demo_presentation.py --stage 3
```

## 七、常见问题

### Q: `Engine file not found` 错误
引擎文件不在安装目录，可能未 `colcon build` 或 `source install/setup.bash`。

```bash
colcon build --packages-select faster_rcnn_ros
source install/setup.bash
bash demo.sh
```

### Q: TRT 推理很慢（>100ms）
未锁频。运行 `sudo jetson_clocks` 后再演示。

### Q: 精度表只显示 "未找到 gt_eval_summary.json"
先运行评估脚本：

```bash
python3 gt_eval_all.py   # 约 2 分钟
bash demo.sh
```

### Q: 无法显示 OpenCV 窗口
无 `$DISPLAY` 时自动跳过，结果保存到 `demo_mosaic.jpg`，用文件管理器查看即可。

### Q: `--stage 2` 推理结束后想立刻看 HTML 报告
`--stage N` 仅运行单个阶段，不生成报告。使用全流程（默认 `--stage 0`）自动生成报告，或手动：

```bash
python3 demo_presentation.py --stage 2 --output /tmp/skip.html
# 报告只有推理相关内容
```

## 八、架构速查

```
用户输入（图片/视频/话题）
    │
    ▼
ROS2节点 (faster_rcnn_node.cpp)
    │  话题: /detectnet/image_in → /detectnet/overlay
    │  文件: image / video / directory 模式
    ▼
检测器 (faster_rcnn_detector.cpp)
    │  blobFromImage → H2D → TRT enqueue → D2H → 过滤
    ▼
TRT引擎 (FP16 114MB / INT8 58MB)
    │  input: [1,3,500,1242]
    │  output: scores / labels / boxes
    ▼
检测结果 vector<Detection>
```

| 精度模式 | 引擎大小 | 延迟均值 | Recall | Precision | 推荐场景 |
|----------|----------|----------|--------|-----------|----------|
| FP16 thr=0.5 | 114MB | 71.4ms | 49.0% | 73.2% | 高精度优先 |
| FP16 thr=0.25 | 114MB | 70.7ms | 72.0% | 31.5% | 召回率优先 |
| INT8 thr=0.5 | 58MB | 62.9ms | 47.8% | 75.4% | **推荐：速度+精度均衡** |
| INT8 thr=0.25 | 58MB | 63.1ms | 73.2% | 28.7% | 召回率+速度优先 |
